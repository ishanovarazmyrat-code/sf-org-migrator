What it does

Everything needed to lift records and their files out of one org and land them in another.

🗂️

Records and files together

Account, Contact, Opportunity and Case out of the box; custom objects with a little config. Files travel with the records they belong to.

📦

Large files

Files up to 2 GB each. Binaries stream to disk and upload in multipart chunks, so a big attachment never has to fit in memory.

🔁

Resumable

Every phase is re-runnable. Progress is tracked in a manifest, so a re-run picks up exactly where it stopped and retries only what failed.

🔗

Relationships preserved

File-to-record links are rebuilt in the target org, resolving each source record to its new target record through external-ID fields.

🕑

Version history (optional)

Migrate just the latest version of each file, or the full version history in order — your choice per run.

🔐

Passwordless login

Authorize each org once through an OAuth device flow. No passwords are stored, and it works the same on a laptop or an unattended VM.

🧭

Automatic field mapping

Copies the fields both orgs share and can write, maps record types by name, and handles state/country codes so inserts don't fail on mismatched values.

🖥️

Terminal or web UI

Drive it from the command line, or open a local web UI with buttons, a live log, and a progress bar. Same engine underneath.

How it works

Four steps, each resumable. Run them in order the first time; re-run any of them to retry.

Connect

Authorize the source and target orgs, then run a preflight check that tells you what's ready and what's missing.

Records

Copy the selected objects into the target, upserting on an external ID and re-pointing lookups to the migrated parents.

Files

Download every file from the source, upload it into the target, and rebuild the links back to the right records.

Verify

See the final counts — documents, versions, and links — plus any failures, so you know the migration is complete.

Quick start

Prepare the target org once, then run the tool.

1 · Install the package in your target org

One click adds the fields, classes, custom setting and permission set the migration needs:
Install into a Salesforce org →  then choose Install for All Users.

2 · Run the tool

# install once, then run from any folder
npm install -g sf-org-migrator

sf-org-migrator login source   # authorize each org (OAuth, no password)
sf-org-migrator login target
sf-org-migrator doctor         # preflight: what's ready, what's missing
sf-org-migrator migrate        # records + files, end to end

Prefer buttons over the terminal? Start the local web UI — setup, object and field selection, and a live progress view:

sf-org-migrator ui   # then open http://localhost:4599
